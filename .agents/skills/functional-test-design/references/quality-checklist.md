# Functional Test Design Quality Checklist

- [ ] Work item change type is classified.
- [ ] Full Mode or Focused Mode is selected intentionally.
- [ ] Source documents are listed.
- [ ] Assumptions are separated from confirmed requirements.
- [ ] Open questions are listed instead of inventing missing requirements.
- [ ] Happy cases cover main flows.
- [ ] Negative cases cover invalid and exceptional conditions.
- [ ] BVA is applied where range constraints exist.
- [ ] EP is applied where input classes exist.
- [ ] Risk-based priority is assigned.
- [ ] Traceability matrix maps requirements to test cases.
- [ ] Expected results are observable and measurable.
- [ ] Non-applicable sections are marked with reason.
- [ ] QA handoff is complete.
